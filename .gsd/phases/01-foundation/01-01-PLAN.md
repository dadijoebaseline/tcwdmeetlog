# Phase 1, Plan 1: Foundation Setup

## Objective

Get a working Next.js + Firebase + Tailwind development environment running locally with basic page structure, database schema, and deployment preview.

## Context

This is the starting point. Without this foundation, no other features can be built. Subsequent phases depend on:
- Firebase Firestore configured and accessible
- Authentication infrastructure ready
- Tailwind CSS working end-to-end
- Next.js running with proper structure

## Tasks

### Task 1: Initialize Next.js Project
**Type:** `auto`

Create a new Next.js 15 project with TypeScript and Tailwind preset.

**Work:**
- Create Next.js app: `npx create-next-app@latest`
- Configure for TypeScript
- Select Tailwind as CSS solution
- Initialize with App Router (not Pages)
- Remove sample files, create basic directory structure

**Verification:**
- [ ] `npm run dev` starts successfully
- [ ] Landing page loads without errors
- [ ] Check `/` route works

---

### Task 2: Configure Firebase
**Type:** `auto`

Set up Firebase project and integrate into Next.js app.

**Work:**
- Create Firebase project in console (free Spark Plan)
- Install Firebase SDK: `npm install firebase`
- Create `/lib/firebase.ts` with Firebase initialization
- Create `.env.local` with Firebase config
- Enable Firestore database (Spark Plan)
- Enable Authentication (Gmail provider - do NOT implement flow yet)
- Test Firestore connection

**Verification:**
- [ ] Firebase console shows project active
- [ ] Firestore database initialized
- [ ] `.env.local` contains credentials
- [ ] Can write/read test document to Firestore

---

### Task 3: Set Up Database Schema
**Type:** `auto`

Design and create Firestore collections for all entities.

**Work:**
- Create schema design document in `.gsd/`
- Collections:
  - `users` (email, role, name, createdAt)
  - `meetings` (title, date, time, venue, createdBy, attendees, status)
  - `attendance` (meetingId, userId, checkInTime, checkOutTime, signature)
  - `qrcodes` (meetingId, code, generatedAt, expiresAt)
- Create Firestore security rules (draft)
- Create indexes for common queries

**Verification:**
- [ ] Schema documented in `.gsd/SCHEMA.md`
- [ ] Collections created in Firestore console
- [ ] Can create test documents

---

### Task 4: Create Base Page Layout
**Type:** `auto`

Build the foundational page layout with navigation.

**Work:**
- Create layouts:
  - `app/layout.tsx` (root with Tailwind globals)
  - `app/(auth)/layout.tsx` (for login, signup)
  - `app/(dashboard)/layout.tsx` (for HR and attendee dashboards)
- Create reusable components:
  - `components/Navbar.tsx`
  - `components/SideNav.tsx` (for dashboard)
  - `components/Button.tsx` (Tailwind styled)
  - `components/Card.tsx` (Tailwind styled)
- Create basic pages:
  - `app/page.tsx` (landing page with project info)
  - `app/(auth)/login.tsx` (stub, will implement in Phase 2)
  - `app/(dashboard)/page.tsx` (stub, redirects based on role in Phase 2)

**Verification:**
- [ ] Landing page renders correctly
- [ ] Tailwind classes applied (colors, spacing, fonts)
- [ ] Navigation links work
- [ ] No TypeScript errors

---

### Task 5: Set Up Environment Variables
**Type:** `auto`

Configure development and production environment variables.

**Work:**
- Create `.env.local.example` with template
- Document all required vars:
  - `NEXT_PUBLIC_FIREBASE_API_KEY`
  - `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN`
  - `NEXT_PUBLIC_FIREBASE_PROJECT_ID`
  - `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET`
  - `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID`
  - `NEXT_PUBLIC_FIREBASE_APP_ID`
  - `NEXT_PUBLIC_APP_ENV` (dev/prod)
- Copy to `.env.local` with actual Firebase values
- Add to `.gitignore`

**Verification:**
- [ ] `.env.local` populated
- [ ] `.env.local.example` has all required vars
- [ ] App reads env vars successfully
- [ ] No credentials exposed in git

**Verification:**

---

### Task 7: Deploy to Firebase Hosting Preview
**Type:** `checkpoint:human-verify`

Build and deploy a preview to Firebase Hosting.

**Work:**
- Install Firebase CLI: `npm install -g firebase-tools`
- Initialize Firebase hosting: `firebase init hosting`
- Build Next.js: `npm run build`
- Deploy to Firebase: `firebase deploy --only hosting`
- Capture preview URL

**Verification:**
- [ ] Build completes without errors
- [ ] Deployment succeeds
- [ ] Preview URL accessible
- [ ] App loads in preview
